# workflows/compile.md — Compilation and ATS Check

Post-processing for applications generated by `workflows/apply-batch.md`.
Mechanical — doesn't touch the tracker (Notion/CSV/none). Optionally takes a
date (`YYYY-MM-DD`); defaults to today if omitted.

**This step never registers the application anywhere.** The tracker is only
touched in `workflows/confirm.md`, after the dev confirms it was actually
sent — this avoids spending an API call (Notion) or a CSV row on an
application that might not even become a real submission.

## 1. Compile + ATS check (delegate to the script)

```bash
python3 scripts/validate_profile.py   # fail fast if the profile is broken
bash scripts/compile-all.sh <date>
```

Compiles each `.tex` referenced in `daily/<date>/*.json` (via `jq`, never
guessing the filename), checks for 1 page, runs the ATS check via `grep`
(see `rules/ats-verification.md`). Mechanical script — doesn't leave a
temporary `.txt` behind, only the verdict.

If a `.tex` fails: read the log, try to fix the obvious cause (unescaped
character, overfull hbox) and recompile once. If it fails again, mark it as
an error in the final report and don't proceed to steps 2-3 for that job.

If the ATS check comes back with issues: "absent gap appears in the CV" is
always blocking; "full/partial stack/gap missing" may be a synonym false
positive — use judgment (see `rules/ats-verification.md`).

## 2. For each successfully compiled job

Read `daily/<date>/<company>.json` (schema in `workflows/apply-batch.md`
§4).

### 2.1 Confirm 1 page

`pdfinfo <pdf> | grep Pages`. Anything other than 1 = treat as a failure,
don't archive, flag for manual intervention (trim content per
`rules/cv-rules.md`).

### 2.2 Archive

Move to `documents/applications/<company>_<role>/` (slugify both):
- `documents/cv/main_<company>.pdf` → `<folder>/main_<company>.pdf`
- `documents/cv/main_<company>.tex` → `<folder>/cv_draft.tex`
- Copy (don't move) `daily/<date>/<company>.json` →
  `<folder>/metadata.json` — this is what `workflows/confirm.md` reads
  later.

Create `<folder>/outcome.md`:

```markdown
# Outcome: <Company> — <Role>

**URL:** <url from the JSON> — submit the CV manually here
**Status:** waiting for send confirmation (run workflows/confirm.md)
**Compilation date:** <date from the JSON>
**Resolution date:** —

## Interview stages reached
- [ ] Phone screen
- [ ] Technical interview
- [ ] System design
- [ ] Final round
- [ ] Offer received

## Notes
```

## 3. Final report

Include the URL for every successfully compiled job:

```
✓ Company X → 1pg ✅ ATS OK ✅ archived → <url>
✗ Company Y → COMPILE ERROR (overfull) → needs manual intervention
8/10 CVs compiled and archived. Go submit at the URLs above, then run
workflows/confirm.md — it's the only step that writes to the tracker.
```
